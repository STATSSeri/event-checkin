# データ保管・廃棄ポリシー

S/PASS（イベント受付システム）におけるユーザーデータの保管期間・廃棄手順を定める。

最終更新：2026-05-19

---

## 1. 対象データと保管期間

### 1-1. 契約期間中

| データ種別 | 保管期間 | 補足 |
|---|---|---|
| ユーザーアカウント情報 | 契約期間中 | アカウント削除まで |
| イベント情報 | 契約期間中 | ユーザーが削除するまで保管 |
| 参加者情報 | 契約期間中 | ユーザーが削除するまで保管 |
| 認証ログ | 90 日 | Supabase Auth 標準 |
| アクセスログ | 90 日 | Vercel 標準 |
| アプリケーションログ | 90 日 | |
| パスワード履歴（ハッシュ） | 直近 10 件 | 11 件目を追加時に古いものを削除 |
| バックアップ | 7 日（Pro）〜30 日（Team） | Supabase の自動バックアップ |
| 請求・領収書情報 | 7 年 | 法定保存期間に基づく |

### 1-2. 個別データの削除

ユーザーは、契約期間中でも以下のデータを任意に削除できます：

- 個別のイベントの削除
- 個別の参加者情報の削除
- 自分のアカウントの削除（要望時、サポート窓口経由）

削除されたデータは、データベースから即時削除され、最大でバックアップ保管期間（7〜30日）経過後に完全消失します。

## 2. 契約終了時の取扱い

### 2-1. ユーザー側のデータ取り出し

契約終了前に、ユーザーは以下の方法でデータを取得できます：

- イベントごとの参加者リスト（CSV エクスポート機能）
- アカウント情報のサポート窓口経由での開示請求

### 2-2. 削除手順

1. **契約終了の確定**：解約申請の受領または契約期間満了
2. **削除予告期間（30 日）**：終了日から 30 日間はデータを保持し、復旧要望に備える
3. **本削除**：30 日経過後、データベースから削除
4. **バックアップからの消失**：本削除から最大 30 日後にバックアップから完全消失

```
[契約終了]──30日──[本削除]──30日──[バックアップから完全消失]
                                  最大60日で復旧不能
```

### 2-3. 削除証明書の発行

ユーザーの要望に応じて、削除完了後に「削除証明書」を電子メールにて発行します。
記載内容：
- ユーザー名（会社名）
- 削除対象データの種別
- 削除完了日時
- 担当者署名

## 3. 法令に基づく保管義務との関係

以下のデータは、法令に基づき本ポリシーの規定にかかわらず一定期間保管します：

| データ | 根拠法令 | 保管期間 |
|---|---|---|
| 請求書・領収書 | 法人税法、消費税法 | 7 年 |
| 取引帳簿 | 会社法 | 10 年 |

法令保管対象データには個人情報が含まれる場合があるため、当該期間中は本ポリシーの削除要請に応じられないことがあります。

## 4. 例外措置

以下の場合、データ削除を一時停止することがあります：

1. 法令・裁判所命令により保全が必要な場合
2. 不正利用・詐欺等の調査が進行中の場合
3. 紛争・係争が発生している場合

該当する場合、当該事象の解決後に速やかに削除を実施します。

## 5. データ削除請求

ユーザーは、自己のアカウントおよび関連データの削除を以下の方法で請求できます：

- 電子メール：[support@statsworks.tokyo](mailto:support@statsworks.tokyo)
- 件名：「データ削除請求 - [会社名／登録メールアドレス]」

請求受領後、本人確認のうえ、合理的な期間（通常 14 日以内）に対応します。

## 6. 参加者からの削除請求

イベント参加者は、原則として当該イベントの主催者（=本サービスのユーザー）に対して削除請求を行うものとします。
当社は、参加者情報を主催者の業務委託先として処理しているため、参加者個人と直接契約関係にありません。

ただし、以下の場合、当社は直接的に対応します：
- 主催者が解約済みで連絡が取れない場合
- 個人情報保護法上の開示・訂正・削除請求権の行使である場合
